# Zeabur 502 / 404 — final checklist

## Your build log shows an OLD image

If `backend-production` only has **5 steps** (no `start-zeabur.sh`, no `chown`), Zeabur used a **cached/old** Dockerfile.

**Fix:** Redeploy → enable **clear cache** / **rebuild without cache** if available, or push commit `2026-05-18-v2` and redeploy.

Latest image must include `start-zeabur.sh` and `ENTRYPOINT []`.

---

## 502 vs 404

| Code | Meaning |
|------|---------|
| **502** | Container not listening (wrong **port**, crash, STARTING) |
| **404** plain text | Zeabur gateway — domain not routed to app, or **no RUNNING** instance |

Right now `devopsdev.zeabur.app` returns **404 from Zeabur** (not `server: uvicorn`) → fix **domain + service binding**, not Django.

---

## Zeabur dashboard (do in order)

### 1. Service `devops` → Overview
- Status must be **RUNNING** (not STARTING / CRASHED)

### 2. Domains tab
- `devopsdev.zeabur.app` must be on service **devops** (not another service)

### 3. Settings
| Field | Value |
|-------|--------|
| Root Directory | `/` OR use `devops.Dockerfile` auto-match |
| Dockerfile override | **empty** |
| Start command | **empty** (use image CMD) |

### 4. Networking
- Port: **8080**
- Variables: **`PORT=8080`** (explicit)

### 5. Variables (required)
```
PORT=8080
DJANGO_CONFIGURATION=Production
PYTHONUNBUFFERED=1
DJANGO_ALLOWED_HOSTS=devopsdev.zeabur.app
DJANGO_SECRET_KEY=<50+ random chars>
DATABASE_URL=<from Zeabur PostgreSQL>
REDIS_URL=<from Zeabur Redis>
```

Use **internal** connection strings from PostgreSQL/Redis service **Connect** tab (not `localhost`).

### 6. Runtime Logs (after redeploy)

Must see within 30 seconds:
```
=== GROW24 Docs backend starting on 0.0.0.0:8080 (workers=1) ===
INFO: Uvicorn running on http://0.0.0.0:8080
```

If **empty logs** → wrong image or instant crash; check Events tab.

### 7. Test (only when logs show Uvicorn)
```bash
curl -I https://devopsdev.zeabur.app/__heartbeat__/
# Expect: HTTP/2 200  and  server: uvicorn
```

### 8. Migrate once
```bash
python manage.py migrate --noinput
```

---

## PostgreSQL + Redis in same project

Without them the app may start but API fails. Add both services in the **same Zeabur project** and link `DATABASE_URL` / `REDIS_URL`.
